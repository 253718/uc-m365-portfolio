## MOP library (Method of Procedure)

Purpose: make change execution predictable by standardizing how procedures are written.

### MOP structure (recommended)

- Preconditions
- Inputs
- Execution steps
- Validation steps
- Evidence capture
- Rollback triggers
- Rollback steps

### Integration with the public repo

This public repo focuses on templates, patterns, and handover artefacts.
Tenant-changing execution lives privately in `uc-m365-ops-change`.
