## Standards

Purpose: define what is considered standard so delivery remains repeatable.

### Architecture standards

- clear responsibility boundaries
- deterministic object naming conventions
- standard call-flow patterns
- read-only validation and evidence capture before change windows

### Delivery standards

- every migration has plan, cutover, rollback, hypercare, and handover artefacts
- outputs follow the public output contract
- evidence packs are produced for handover and audit support
- quality gates include CI, tests, static analysis, and read-only guardrails
